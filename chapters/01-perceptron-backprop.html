Chapter 1

The Perceptron & Backpropagation

The idea that computation could model learning predates digital computers. In 1943, McCulloch and Pitts described a binary model of the neuron. Frank Rosenblatt's 1958 perceptron built on this with the first learning algorithm: a single layer of weighted connections adjusted based on prediction errors. Given labeled examples, the perceptron iteratively updates its weights until it correctly classifies the training data — provided the data is linearly separable. The proof of convergence, which Rosenblatt provided, was the first formal guarantee that a machine could learn from experience.

The limitation is the constraint. A perceptron can only separate data with a straight line (or hyperplane) through input space. Minsky and Papert formalized this in 1969, showing it could not even compute XOR, a simple non-linear function. Their critique contributed to a prolonged reduction in interest and funding for neural networks. The core problem they identified was real: multi-layer networks could represent non-linear functions, but there was no known way to train them. The weights in intermediate layers received no direct error signal.

Rumelhart, Hinton, and Williams' 1986 Nature paper answered this. Applying the chain rule of calculus, backpropagation efficiently computes the gradient of a loss function with respect to every weight in a multi-layer network by propagating error signals backward from the output. Each weight's update is proportional to how much it contributed to the error. The algorithm is not conceptually new — versions had been described earlier — but the 1986 paper demonstrated it clearly, applied it to interesting problems, and showed that hidden layers learn meaningful internal representations of data rather than arbitrary encodings. This made the case that depth was not just mathematically necessary for expressiveness but practically useful for generalization.

Backpropagation remains the training algorithm for every neural network described in the chapters that follow. The papers in this chapter establish the two ideas that make it possible: that networks can learn by following gradients, and that layered architectures can represent functions that no single layer could.
