Chapter 8

Retrieval and Tool Use

A language model's knowledge is encoded in its parameters at training time. This creates two structural limitations. First, the knowledge is static: a model trained on data through a certain date cannot know what happened afterward. Second, the knowledge is uncited: when a model produces a factual claim, there is no mechanism to identify which training examples support it or whether they do at all. Retrieval-augmented generation and tool use represent complementary responses to these constraints.

Lewis, Perez, Piktus, Petroni, Karpukhin, Goyal, Küttler, Lewis, Yih, Rocktäschel, Riedel, and Kiela's Retrieval-Augmented Generation (RAG, 2020) combined a dense retrieval model with a sequence-to-sequence generator. Given an input query, a retrieval component — a dense passage retrieval model — retrieves relevant documents from an external knowledge base; the generator then produces output conditioned on both the query and the retrieved passages. Unlike fine-tuning, retrieval updates are immediate: changing what documents are indexed changes what the model can reference without retraining. RAG improved performance substantially on knowledge-intensive tasks such as open-domain question answering, and the architecture made generation more grounded — outputs could be traced to specific retrieved passages rather than attributed only to opaque parametric memory.

Yao, Zhao, Yu, Taylor, Shafran, Narasimhan, and Cao's ReAct (2023) extended the idea from passive retrieval to active tool use. A ReAct agent interleaves reasoning steps — expressed as natural language "thoughts" — with actions (search queries, API calls, code executions) within a single generation. Rather than retrieving context before reasoning begins, the agent reasons about what information it needs, takes an action to obtain it, observes the result, updates its reasoning, and continues. This interleaving grounds the reasoning process in evidence at each step rather than only at the start. ReAct substantially outperformed both reasoning-only and action-only baselines on knowledge-intensive tasks, and the reasoning traces made failure diagnosis tractable: when the agent made an error, the trace showed which step went wrong and why.

Together, RAG and ReAct define the basic architecture of knowledge-grounded language model systems: retrieval provides relevant context from a static index; tool use extends the agent's reach to dynamic information not represented in any fixed retrieval corpus. Subsequent work — including Toolformer's learning of when and how to call APIs, and WebGPT's supervised training of a web-browsing agent — has extended these ideas further. Contemporary AI agents are largely elaborations of this retrieval-and-action loop, extended with longer contexts, more capable tools, and better integration with persistent state across multi-step tasks.
